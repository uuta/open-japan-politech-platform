# 政治資金データ収集調査レポート

## 概要

日本の全政党の政治資金データを体系的に収集するための調査結果をまとめる。総務省の公開データ、都道府県選管のデータ、既存の民間プロジェクト、技術的な収集方法、法的観点を網羅的に整理する。

---

## 1. 総務省の政治資金収支報告書

### 1.1 公開元

- **URL**: https://www.soumu.go.jp/senkyo/seiji_s/seijishikin/
- **管轄**: 総務大臣届出分の政治団体（政党本部、政治資金団体、国会議員関係政治団体等）
- **公表時期**: 原則として毎年11月30日までに定期公表。追加分は随時公表
- **最新公表**: 令和7年11月28日公表（令和6年分 定期公表）

### 1.2 データフォーマット

| 項目 | 状況 |
|------|------|
| **PDF** | 主要なフォーマット。スキャンPDFとテキストPDFが混在 |
| **XML** | 現時点では提供なし |
| **CSV** | 現時点では提供なし |
| **API** | 現時点では提供なし |
| **データベース** | 令和10年4月1日までに開始予定（改正法に基づく） |

### 1.3 データの構造

総務省サイトでの分類カテゴリ:

1. **政党本部**（SF）
2. **政党の支部**（SL）
3. **政治資金団体**（SC）
4. **国会議員関係政治団体**（SS）
5. **その他の政治団体**（SO）
6. **資金管理団体**（SG）

各報告書のURL構造:
```
https://www.soumu.go.jp/senkyo/seiji_s/seijishikin/reports/SS{公表日}/{カテゴリ}/{番号}.html
```

### 1.4 報告書の主な記載項目

- 収入総額（寄附、事業収入、政党交付金、借入金等）
- 支出総額（経常経費、政治活動費）
- 資産等の状況
- 寄附者（個人・法人）の名前、住所、金額
- 政治資金パーティーの収入・対価支払者
- 政治活動費の支出先・目的

### 1.5 保存期間

- **収支報告書**: 公開から **3年間** で削除（政治資金規正法第20条の2）
- **政党交付金使途等報告書**: 公開から **5年間**

### 1.6 制限事項

- PDFの印刷がブロックされているケースがある（政党交付金使途等報告書）
- 3年経過後はウェブ上から削除され、原本も廃棄される
- スキャンPDF（画像）とテキストPDFが混在し、統一されていない

## 2. 都道府県選挙管理委員会

### 2.1 概要

- 都道府県知事届出分の政治団体は各都道府県の選挙管理委員会が管轄
- **47都道府県中46都道府県**がウェブで公開（新潟県のみウェブ非公開、情報公開請求で取得可能）
- 地方議員、市区町村長の関係政治団体は都道府県選管が管轄

### 2.2 公開フォーマットの状況

| 分類 | 状況 |
|------|------|
| **フォーマット** | 全都道府県ともPDF形式が主流 |
| **統一規格** | なし。各都道府県で独自のフォーマット |
| **公開時期** | 都道府県により異なる（多くは11月頃） |
| **保存期間** | 3年間（法律に基づく） |

### 2.3 都道府県間の差異

- PDF生成方法が異なる（テキストPDF vs スキャンPDF）
- ファイル命名規則が統一されていない
- サイト構造・URL構造がバラバラ
- 一部の都道府県は手書きの報告書をスキャンしたもの

## 5. 技術的な収集方法

### 5.1 PDFからのデータ抽出

#### テキストPDFの場合

| ツール | 言語 | 特徴 |
|--------|------|------|
| **Camelot** | Python | テーブル抽出に強い。精度メトリクスあり |
| **Tabula / tabula-py** | Python/Java | 使いやすい。DataFrame出力 |
| **pdfplumber** | Python | テキスト+テーブル抽出 |
| **PyMuPDF (fitz)** | Python | 高速なテキスト抽出 |

#### スキャンPDF（画像）の場合

| ツール | 特徴 |
|--------|------|
| **Tesseract OCR (pytesseract)** | オープンソースOCR。日本語対応（lang='jpn'） |
| **pdf2image + Tesseract** | PDF→画像→OCRのパイプライン |
| **LangChain + LLM** | Polimoneyで採用。AI による構造化データ抽出 |
| **Google Cloud Vision API** | 高精度な日本語OCR |
| **Amazon Textract** | テーブル抽出に特化 |

#### 課題

- 手書き報告書の認識精度が低い
- 漢字の揺れ（人名・社名）による誤認識
- 数値の誤読（特に金額データ）
- 報告書フォーマットが統一されていない（PC作成 vs 手書き）
- PDFが90度回転しているケースがある

### 5.2 XMLデータの有無

- **現時点ではXMLデータは提供されていない**
- 改正法により令和9年（2027年）からオンライン提出が義務化されるが、提出フォーマットの詳細は未確定
- 総務省の「政治資金関係申請・届出オンラインシステム」（https://kyoudou.soumu.go.jp/）が存在するが、データの公開形式は別問題

### 5.3 定期的な自動収集の仕組み（提案）

```
[データ収集パイプライン]

1. スクレイピング層
   ├── 総務省サイトのPDFリンク収集（年1回定期公表 + 追加分）
   ├── 47都道府県選管サイトの巡回
   └── 新規公表の検知（差分チェック）

2. PDF処理層
   ├── テキストPDF判定
   │   ├── テキスト抽出（pdfplumber / PyMuPDF）
   │   └── テーブル抽出（Camelot / tabula-py）
   ├── スキャンPDF判定
   │   ├── 画像変換（pdf2image）
   │   ├── OCR（Tesseract / Cloud Vision API）
   │   └── AI構造化（LangChain / LLM）
   └── 品質チェック（金額の合計検証、文字認識の信頼度）

3. 正規化層
   ├── 政治団体名の名寄せ
   ├── 人名・法人名の統一
   ├── 金額フォーマットの統一
   ├── 日付フォーマットの統一
   └── カテゴリの標準化

4. ストレージ層
   ├── 原本PDF保存（アーカイブ）
   ├── 構造化データ保存（PostgreSQL / BigQuery）
   ├── 検索インデックス（Elasticsearch）
   └── API提供（REST / GraphQL）
```

## 6. 法的な観点

### 6.1 公開義務

- **政治資金規正法**により、政治団体は収支報告書の提出が義務付けられている
- 総務大臣または都道府県知事が報告書を公表する義務がある
- 公表方法: インターネット掲載（平成20年改正以降）
- 法的根拠: 政治資金規正法 第20条の2、第29条等

### 6.2 改正政治資金規正法（令和6年6月・12月改正）のスケジュール

| 施行日 | 内容 |
|--------|------|
| **令和8年（2026年）1月1日** | 代表者責任強化、政治資金監査拡充、パーティー対価支払の口座振込義務化、インターネット公表開始 |
| **令和9年（2027年）1月1日** | 政党本部・政治資金団体・国会議員関係政治団体の **オンライン提出義務化**、パーティー公開基準額を20万円超→5万円超に引下げ |
| **令和10年（2028年）4月1日** | **データベースを用いた公表** の運用開始（令和8年定期公表分以降が対象） |

### 6.3 二次利用の法的制約

| 観点 | 状況 |
|------|------|
| **著作権** | 政治資金収支報告書は行政文書・公文書に準ずるものであり、著作権法第13条により「国・地方公共団体の告示、通達等」は著作権の対象外。ただし報告書自体は政治団体が作成するため、厳密な解釈は分かれる |
| **二次利用** | 法律で明示的に禁止する規定はない。ただし総務省は政党交付金使途等報告書の印刷をブロックしている |
| **再配布** | 法律上の明示的禁止はないが、総務省の利用規約による制限の可能性あり |
| **個人情報** | 寄附者の住所は改正法により「都道府県、郡及び市町村の名称」に限定される（令和8年以降） |
| **保存期間の問題** | 3年で廃棄されるため、アーカイブの法的根拠が問題になりうる |

### 6.4 実務的な見解

- 政治資金センター、政策推進機構DB、政治資金データベース等、複数の民間プロジェクトが既にデータの二次利用・公開を行っており、法的に問題視された事例は確認されていない
- 政治資金規正法第1条は「政治活動が国民の不断の監視と批判の下に行われるようにする」ことを目的としており、二次利用はこの趣旨に合致する
- 総務省のデータは公表を前提としたものであり、オープンデータとしての再配布は法的リスクが低いと考えられる

## 7. 収集戦略の推奨

### 7.1 短期（即座に実行可能）

1. **既存データベースの活用**: 政策推進機構DB / 政治資金センターのデータを基盤として活用
2. **Polimoney との連携**: AGPL-3.0ライセンスのオープンソースプロジェクトと協力・フォーク
3. **総務省PDFの直接ダウンロード**: 現在公開中の3年分のデータを取得・保存

### 7.2 中期（数ヶ月〜1年）

1. **独自OCRパイプラインの構築**: LangChain + LLM を活用した高精度なデータ抽出
2. **都道府県選管データの体系的収集**: 46都道府県 + 新潟県（情報公開請求）
3. **データアーカイブの構築**: 3年で削除されるデータの永続保存

### 7.3 長期（1年以上）

1. **改正法に基づくデータベースとの連携**: 2028年4月開始予定の総務省公式DBとの統合
2. **API の整備**: REST / GraphQL API によるデータ提供
3. **国際連携**: 海外の政治資金透明化プロジェクト（OpenSecrets等）との比較・連携
